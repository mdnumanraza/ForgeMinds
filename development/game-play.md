# BOSS SYSTEM

At the end of stages/worlds:
player encounters a Boss Gatekeeper.

Bosses should:
- challenge understanding
- combine multiple concepts
- test practical knowledge
- require score thresholds

Boss mechanics can include:
- MCQs
- code challenges
- sequence puzzles
- debugging tasks
- practical mini challenges

To pass:
player must achieve:
- >= 75% score

If failed:
player is NOT simply blocked.

Instead:
- redirected into exploration
- encouraged to gather more knowledge
- discover hidden learning content
- complete side quests
- strengthen understanding

This creates:
- RPG progression loop
- natural reinforcement learning
- replayability
