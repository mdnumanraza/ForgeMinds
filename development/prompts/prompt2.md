# PROJECT OVERVIEW

ForgeMinds is a story-driven RPG learning game.

Players learn technical topics through exploration, quests, dungeons, NPC interactions, combat, and boss battles.

The first game is:

Kubernetes Kingdom

The first two themes are:

1. Fantasy Kingdom
2. Space Galaxy

The learning content remains identical.

Only visuals, story, NPCs, enemies, locations, and lore change.

---

# CORE DESIGN PRINCIPLE

Knowledge must become gameplay.

Learning should not feel separate from gameplay.

The player should not feel like:

"I am studying."

The player should feel like:

"I am on an adventure."

Learning should naturally happen while progressing through the story.

---

# KUBERNETES KINGDOM STORY

The player is summoned to Kubernetes Kingdom.

A powerful dragon has attacked the kingdom.

The attack caused chaos:

* Pods were destroyed
* Nodes were damaged
* Communication broke down
* Security weakened
* Services stopped functioning
* The kingdom became unstable

The player is chosen as the hero.

To save the kingdom, the player must:

* learn Kubernetes concepts
* restore damaged systems
* help villagers
* solve problems
* defeat corrupted creatures
* eventually defeat the dragon

The dragon represents the final challenge.

The player becomes stronger by gaining Kubernetes knowledge.

# KNOWLEDGE DISCOVERY

The player discovers knowledge while exploring.

Examples:

* ancient scrolls
* wisdom stones
* knowledge tablets
* books
* magical crystals
* NPC conversations

When interacted with:

A learning panel appears containing:

* simple explanation
* real-world analogy
* technical definition
* examples
* commands
* practical use cases
* important notes

Learning content should be broken into small chunks.

Never giant walls of text.

---

# QUEST SYSTEM

Example:

NPC:

"The Pod Bug has infected the village."

Quest:

Learn about Pods.

Player explores.

Finds Pod knowledge scrolls.

Learns the concept.

Encounters Pod Bug.

Defeats Pod Bug using knowledge.

Returns to NPC.

Quest completed.

Reward:

* XP
* coins
* items
* progression

---

# KNOWLEDGE COMBAT SYSTEM

Combat is NOT traditional combat.

Knowledge powers combat.

Player encounters enemy.

Enemy presents challenges.

Examples:

* MCQs
* command completion
* code recognition
* scenario questions
* debugging questions
* matching exercises
* ordering exercises

Avoid:

* essays
* long text answers
* subjective answers

Only deterministic answers.

---

# COMBAT FLOW

Player encounters enemy.

Enemy challenge appears.

Correct answers charge abilities.

Example:

2/2 correct:

* full sword charge
* critical attack

1/2 correct:

* partial charge
* weaker attack

0/2 correct:

* attack fails

Knowledge directly affects combat performance.

---

# PLAYER FAILURE

Failure should not feel punishing.

Avoid:

Wrong answer
→ Immediate death

Prefer:

Wrong answer
→ Reduced HP
→ Enemy survives
→ Retry opportunity

Learning should encourage experimentation.

---

# DUNGEONS

Each stage contains a dungeon.

Dungeon purpose:

* deeper learning
* tougher enemies
* hidden knowledge
* mini-bosses
* advanced concepts

Dungeon completion unlocks the stage boss.

---

# BOSS DESIGN

Bosses validate mastery.

Bosses combine multiple concepts from the stage.

Examples:

* MCQs
* command questions
* scenario questions
* debugging tasks

Bosses should feel different from normal enemies.

Bosses should require understanding, not memorization.

---

# BOSS COMBAT IDEA

Boss attacks.

Player selects action:

* sword
* shield
* spell
* special ability

Abilities are powered by knowledge.

Example:

Boss launches attack.

Player chooses shield.

Shield challenge appears.

Correct answer:

Shield activates.

Wrong answer:

Shield fails.

Boss deals damage.

Then player attacks.

Attack power depends on answer quality.

Continue until:

* Boss HP reaches zero
  OR
* Player HP reaches zero

---

# PROGRESSION SYSTEM

Player gains:

* XP
* levels
* coins
* equipment
* abilities

Campaign progression is independent.

Example:

Kubernetes Kingdom:
Level 12

Linux Kingdom:
Level 4

Docker Kingdom:
Level 7

Future games should not share progression.
